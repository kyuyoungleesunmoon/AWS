# AWS 입문자를 위한 상세 실습 가이드 (2025년 기준)

이 가이드는 AWS 클라우드 서비스를 처음 접하는 사용자를 위해 작성되었습니다. **계정 보안(IAM)**, **가상 서버(EC2)**, **데이터베이스(RDS)** 순서로 진행하며, 각 단계는 마우스 클릭 하나까지 상세하게 설명합니다.

**작성 기준일**: 2026년 1월 2일 (최신 콘솔 UI 반영)
**준비물**:
1.  AWS 계정 (루트 계정 로그인 상태)
2.  Windows PC (PowerShell 사용)

---

## 1. AWS IAM: 관리자 사용자 만들기
**왜 필요한가요?**
루트 계정(이메일로 로그인하는 계정)은 모든 권한을 다 가지고 있어 해킹당하면 돌이킬 수 없습니다. 따라서 평소에는 권한을 제한하거나 관리가 가능한 'IAM 사용자'를 만들어 사용하는 것이 원칙입니다.

### 1-1. IAM 대시보드 접속
1.  AWS 콘솔 상단 **검색창(Alt+S)**에 `IAM`을 입력합니다.
2.  서비스 목록에서 **IAM**을 클릭합니다.

### 1-2. 사용자 생성
1.  왼쪽 메뉴바에서 **[사용자]**를 클릭합니다.
2.  우측 상단의 주황색 버튼 **[사용자 생성]**을 클릭합니다.
3.  **1단계: 사용자 세부 정보 지정**
    *   **사용자 이름**: `admin-user` (또는 본인 영문 이니셜) 입력.
    *   **AWS Management Console에 대한 사용자 액세스 권한 제공**: 체크박스 **[v] 체크**.
    *   **콘솔 암호**: '사용자 지정 암호' 선택 후 기억하기 쉬운 암호 입력.
    *   **사용자는 다음 로그인 시 새 암호를 생성해야 합니다**: 실습 편의를 위해 **[ ] 체크 해제**.
    *   **[다음]** 클릭.
4.  **2단계: 권한 설정**
    *   **권한 옵션**: 3번째 박스인 **[직접 정책 연결]** 선택.
    *   **권한 정책**: 아래 검색창에 `AdministratorAccess` 입력 후 엔터.
    *   검색 결과 목록 맨 위에 있는 `AdministratorAccess` 왼쪽 체크박스 **[v] 체크**.
    *   **[다음]** 클릭.
5.  **3단계: 검토 및 생성**
    *   설정 내용을 확인하고 우측 하단 **[사용자 생성]** 클릭.

### 1-3. 로그인 정보 저장 (매우 중요)
1.  '사용자를 검색했습니다' 화면이 나옵니다.
2.  **콘솔 로그인 URL**: `https://<숫자>.signin.aws.amazon.com/console` 형태의 주소를 복사하여 메모장이나 브라우저 즐겨찾기에 저장합니다.
3.  **[csv 파일 다운로드]** 버튼을 눌러 계정 정보를 내 PC에 저장해 둡니다.
4.  이제 현재 브라우저에서 로그아웃하거나, **새 시크릿 창**을 열어 위 URL로 접속한 뒤 `admin-user`로 로그인합니다. (이후 모든 실습은 `admin-user`로 진행)

## 2. AWS EC2: 가상 서버(컴퓨터) 만들기
**왜 필요한가요?**
클라우드 상에 나만의 컴퓨터(Linux 서버)를 한 대 임대하는 것입니다.

### 2-1. EC2 대시보드 및 리전 설정
1.  콘솔 우측 상단의 지역 이름(예: 버지니아 북부)을 클릭하여 **[아시아 태평양(서울) ap-northeast-2]**를 선택합니다.
2.  상단 검색창에 `EC2` 입력 후 접속합니다.
3.  중앙의 주황색 버튼 **[인스턴스 시작]**을 클릭합니다.

### 2-2. 인스턴스 설정 (상세)
1.  **이름 및 태그**:
    *   **이름**: `MyFirstWebServer` 입력.
2.  **애플리케이션 및 OS 이미지(AMI)**:
    *   `Amazon Linux` 선택 (기본값).
    *   **AMI**: `Amazon Linux 2023 AMI` 확인 (프리 티어 사용 가능 태그 확인).
3.  **인스턴스 유형**:
    *   `t2.micro` 또는 `t3.micro` 선택 (프리 티어 사용 가능 태그 확인).
4.  **키 페어(로그인)** (가장 중요!):
    *   우측의 **[새 키 페어 생성]** 텍스트 링크 클릭.
    *   **키 페어 이름**: `my-ec2-key` 입력.
    *   **키 페어 유형**: `RSA` 선택.
    *   **프라이빗 키 파일 형식**: `.pem` 선택.
    *   **[키 페어 생성]** 버튼 클릭.
    *   **확인**: 브라우저가 자동으로 `my-ec2-key.pem` 파일을 다운로드합니다. 내 PC의 `다운로드(Downloads)` 폴더를 확인하세요. 이 파일은 서버의 열쇠이므로 절대 잃어버리면 안 됩니다.
5.  **네트워크 설정**:
    *   우측 상단 **[편집]** 버튼 클릭.
    *   **보안 그룹 이름**: `web-server-sg` 등으로 변경.
    *   **인바운드 보안 그룹 규칙**:
        *   유형: `ssh`
        *   소스 유형: `위치 무관(0.0.0.0/0)` -> 보안을 위해 **[내 IP]**로 변경 권장.
6.  **스토리지 구성**: 기본값(8 GiB, gp3) 유지.
7.  우측 요약 패널 하단의 **[인스턴스 시작]** 클릭.

## 3. 내 PC(Windows)에서 EC2 접속하기
**목표**: 방금 다운로드한 키 파일을 이용해 내 컴퓨터의 검은 화면(터미널)에서 AWS 서버로 접속합니다.

### 3-1. 키 파일 준비
1.  파일 탐색기(`Win+E`)를 엽니다.
2.  `내 PC` -> `로컬 디스크 (C:)`로 이동하여 `AWS`라는 새 폴더를 만듭니다 (`C:\AWS`).
3.  `다운로드` 폴더에 있는 `my-ec2-key.pem` 파일을 복사하여 `C:\AWS` 폴더 안에 붙여넣습니다. (최종 경로: `C:\AWS\my-ec2-key.pem`)

### 3-2. 터미널(PowerShell) 실행
1.  키보드의 `Windows` 키를 누르고 `PowerShell`을 입력합니다.
2.  나오는 아이콘에 마우스 우클릭 -> **[관리자 권한으로 실행]** 클릭.

### 3-3. 키 파일 권한 설정 (필수)
AWS는 키 파일이 너무 공개되어 있으면 접속을 차단합니다. 아래 명령어를 복사해서 PowerShell에 붙여넣고 엔터(Enter)를 치세요.

```powershell
# 1. 상속된 권한 제거
icacls.exe "C:\AWS\my-ec2-key.pem" /reset

# 2. 현재 사용자에게만 읽기 권한 부여 (나 외에 아무도 못 쓰게 설정)
icacls.exe "C:\AWS\my-ec2-key.pem" /grant:r "$($env:USERNAME):(R)" /inheritance:r
```
*성공 시 `처리된 파일: C:\AWS\my-ec2-key.pem 1개` 같은 메시지가 뜹니다.*

### 3-4. 접속 명령어 입력
1.  AWS 콘솔로 돌아와 실행 중인 `MyFirstWebServer` 인스턴스를 클릭합니다.
2.  화면 중앙 아래쪽 상세 정보 탭에서 **[퍼블릭 IPv4 주소]** (예: 3.12.34.56)를 찾아 복사 아이콘을 누릅니다.
3.  PowerShell에 아래 명령어를 입력합니다. (`<IP주소>` 부분은 지우고 복사한 IP를 붙여넣으세요)

```powershell
ssh -i "C:\AWS\my-ec2-key.pem" ec2-user@<여기에-복사한-IP주소-붙여넣기>
```

4.  엔터를 치면 `Are you sure... (yes/no)?` 라고 묻습니다. `yes` 라고 타이핑하고 엔터를 칩니다.
5.  화면에 `[ec2-user@ip-172-31-...]` 같은 글자가 뜨면 접속 성공입니다!

## 4. AWS RDS: 데이터베이스 연결하기
**목표**: 관리형 데이터베이스를 만들고, 위에서 만든 EC2 서버가 이 DB를 사용할 수 있게 연결합니다.

### 4-1. RDS 생성 마법사
1.  상단 검색창에 `RDS` 입력 후 접속.
2.  **[데이터베이스 생성]** 주황색 버튼 클릭.
3.  **생성 방식**: `표준 생성` 선택.
4.  **엔진 옵션**: `MySQL` 선택.
5.  **템플릿**: **[프리 티어]** 선택 (이걸 안 하면 요금이 나옵니다. 중요!).
6.  **설정**:
    *   **DB 인스턴스 식별자**: `mydb`
    *   **마스터 사용자 이름**: `admin`
    *   **마스터 암호**: 접속 비번 입력 (예: `password1234` - 실제론 복잡하게!). 암호 확인에도 똑같이 입력.
7.  **인스턴스 구성**: `db.t3.micro` (프리 티어) 확인.

### 4-2. 연결 설정 (가장 쉬운 방법)
이 부분이 핵심입니다. AWS가 복잡한 네트워크 설정을 대신해 줍니다.

1.  **연결** 섹션 찾기.
2.  **컴퓨팅 리소스**: **[EC2 컴퓨팅 리소스에 연결]** 선택.
3.  **EC2 인스턴스**: 드롭다운을 눌러 아까 만든 `MyFirstWebServer` 선택.
4.  나머지는 기본값 유지.

### 4-3. 생성 완료
1.  페이지 맨 아래로 내려가 **[데이터베이스 생성]** 클릭.
2.  생성이 시작되면 상태가 `생성 중`으로 뜹니다. 약 5~10분 후 `사용 가능`으로 바뀌면 완료된 것입니다.

## 5. 실습 종료 및 자원 삭제 (과금 방지)
공부를 마쳤으면 반드시 삭제해야 돈이 안 나갑니다.

1.  **EC2 삭제**: EC2 콘솔 -> 인스턴스 선택 -> [인스턴스 상태] -> **[인스턴스 종료(삭제)]**.
2.  **RDS 삭제**: RDS 콘솔 -> 데이터베이스 선택 -> [작업] -> **[삭제]** -> '최종 스냅샷 생성' 체크 해제 -> '삭제하겠습니까?' 확인 문구 입력 -> **[삭제]**.
